## Full native rebuild (after adding new native packages or running pod install)

### Step 1 — Re-create header shims (REQUIRED after pod install)
`expo-av` header removal from `ExpoModulesCore` XCFramework requires ObjC shims:
```bash
for slice in ios-arm64 ios-arm64_x86_64-simulator; do
  HDR="/Users/cguzman/Documents/personal/app/myManager/ios/Pods/ExpoModulesCore/ExpoModulesCore.xcframework/${slice}/ExpoModulesCore.framework/Headers"
  printf '#import <Foundation/Foundation.h>\n@protocol EXEventEmitter <NSObject>\n- (NSArray<NSString *> *)supportedEvents;\n- (void)startObserving;\n- (void)stopObserving;\n@end\n' > "${HDR}/EXEventEmitter.h"
  printf '#import <Foundation/Foundation.h>\n@protocol EXEventEmitterService <NSObject>\n- (void)sendEventWithName:(NSString *)eventName body:(id)body;\n@end\n' > "${HDR}/EXEventEmitterService.h"
  printf '#import <Foundation/Foundation.h>\n@protocol EXLegacyExpoViewProtocol <NSObject>\n@optional\n- (void)setModuleRegistry:(id)moduleRegistry;\n@end\n' > "${HDR}/EXLegacyExpoViewProtocol.h"
  echo "Shims created for ${slice}"
done
```
The `ios/Podfile` post_install hook also does this automatically on future pod installs.

### Step 2 — Build and run
```bash
cd /Users/cguzman/Documents/personal/app/myManager
export DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer
npx expo run:ios --device "iPhone 17 Pro"
```
This runs: prebuild → pod install → xcodebuild → simulator launch → Metro start.

**Expected duration:** ~3 min incremental (pods cached), ~8 min first time.

## Troubleshooting

### Metro transformer crash: `Cannot find module 'babel-preset-expo'`
```bash
npm install babel-preset-expo --legacy-peer-deps
```
Then kill Metro and restart with `--clear`.

### `Cannot find native module 'ExpoLinking'`
`expo-linking` wasn't in `ExpoModulesProvider.swift`. Fix:
```bash
cd /Users/cguzman/Documents/personal/app/myManager/ios
pod install
# Then re-create header shims (see above)
# Then rebuild: npx expo run:ios --device "iPhone 17 Pro"
```

### Blank white screen (no error)
Check `package.json` `main` field — must be `"expo-router/entry"` not `"index.ts"`.

### `react-native-worklets` build error: `Build input file cannot be found`
The package must be installed explicitly (peer dep of react-native-reanimated 4.x):
```bash
npm install react-native-worklets@0.8.3 --legacy-peer-deps
```

### Stale DerivedData causing old file path errors
```bash
rm -rf ~/Library/Developer/Xcode/DerivedData/myManager-*
```
Then rebuild normally.

### `xcrun simctl` not found
```bash
export DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer
```
Must be set in every shell session. Add to `.zshrc` or `.bash_profile` to persist.

### npm install fails with peer dep conflicts
Always use `--legacy-peer-deps`:
```bash
npm install <package> --legacy-peer-deps
```
Or use `npx expo install <package>` which handles Expo-specific versioning.
